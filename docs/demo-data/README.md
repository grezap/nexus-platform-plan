# Demo Data — Seed-Data Kit

Every demo scenario runs against realistic synthetic datasets. The datasets are produced by **seeded, deterministic generators** checked into this repository; CI verifies that every generator produces bit-identical output on re-run so demos stay reproducible.

## Datasets

### Retail domain

- **Volumes** — 10,000 customers · 50,000 products · 500,000 orders · ~2M order lines · 100K shipments · 5K warehouse rows.
- **Generators** —
  - `demo-data/generators/retail.cs` (.NET 10 + **Bogus** 35+) — produces CSV + Parquet partitioned by month. Seed `42`.
  - `demo-data/generators/retail.py` (Python 3.13 + **Faker** 30+ + **Polars**) — same schema, same RNG seed; outputs bit-identical Parquet (verified in CI by SHA-256 of the sorted Parquet pages).
- **Profiles** — `small` (1K/5K/50K), `medium` (10K/50K/500K default), `large` (100K/500K/5M), `showcase` (curated rows hand-picked for visually compelling demos).
- **Used by** — DEMO-01, DEMO-03, DEMO-07, DEMO-12, DEMO-14.

### Financial domain

- **Volumes** — 50 symbols · 5 years · 1-minute OHLCV · ~26M rows.
- **Generator** — `demo-data/generators/finance.py`. Geometric Brownian Motion with configurable volatility clusters and deterministic jump-diffusion events. Seed `42`. Output: Iceberg-compatible Parquet partitioned by symbol + year.
- **Profiles** — `1y-10sym`, `5y-50sym` (default), `fraud-fixture` (embeds 20 labelled anomalies for SentinelML).
- **Used by** — DEMO-02, DEMO-06, DEMO-08, DEMO-13.

### Document corpus (LocalMind RAG)

- **Sources** — the 14 Volume `.txt` files from `../DOCS/_build/txt/` (canonical) plus 200 synthetic enterprise KB articles generated by `demo-data/generators/kb-corpus.py`.
- **Output** — text chunks with paragraph boundaries preserved, front-matter containing volume + section anchors for citation.
- **Used by** — DEMO-04, DEMO-14.

### Images (VisionCore)

- **Volumes** — 1,000 synthetic product photos · 500 defect samples with labelled bounding boxes.
- **Generator** — `demo-data/generators/vision.py` builds product photos via deterministic procedural rendering (Pillow + NumPy) and overlays defects (cracks, dents, discolorations) with known ground truth.
- **Storage** — `demo-data/fixtures/vision/` tracked via **Git LFS**.
- **Profiles** — `defect-showcase` (50 paired good/defect), `full` (1,500 images).
- **Used by** — DEMO-05.

### Telemetry (QueryLens)

- **Generator** — `demo-data/generators/querylens-workload.py` produces a synthetic SQL Server workload — parametrized batches with configurable skew that causes known plan-hash regressions when statistics drift. Outputs a `.sqlreplay` file consumable by the `querylens` demo harness.
- **Profiles** — `baseline`, `regression-fixture` (the pre-seeded regression used in DEMO-09).
- **Used by** — DEMO-09.

### Time-series (ChronoSight, SentinelML)

- **Generator** — `demo-data/generators/timeseries.py`. Regime-configurable (trend / seasonality / volatility / change-points) with a YAML scenario file controlling regime timing. Produces Kafka-ingestable JSON and Parquet snapshots. Built on Polars + NumPy.
- **Profiles** — `steady`, `regime-shift`, `seasonal`, `anomaly-cluster`.
- **Used by** — DEMO-02, DEMO-06, DEMO-13.

### Interactions (RecoEngine)

- **Generator** — `demo-data/generators/reco.py`. Simulated user sessions with configurable affinity profiles (outdoor, electronics, household, etc.). Generates click / view / purchase events into Kafka.
- **Profiles** — `showcase` (curated personas for predictable demo outcomes), `broad` (uniform).
- **Used by** — DEMO-07.

## Reproducibility

- **Seeds.** Every generator accepts `--seed` (default 42) and fails loudly if the seed produces different output than the snapshot fixture.
- **Determinism checks.** CI runs each generator twice and SHA-256-diffs the outputs (after canonical sorting for unordered formats like Parquet row groups).
- **Cross-runtime parity.** Where both .NET and Python generators exist for the same dataset (retail), a CI job runs both and asserts byte-identical Parquet.
- **Fixture snapshots.** `demo-data/fixtures/` holds the expected SHA-256 hashes per profile per generator.
- **No external network.** Generators must not hit external services during execution; an offline CI runner is used.

## CLI entry points

```
nexus-cli seed retail --profile=medium
nexus-cli seed finance --profile=5y-50sym
nexus-cli seed vision --set=defect-showcase
nexus-cli seed querylens --profile=regression-fixture
nexus-cli seed reco --profile=showcase
nexus-cli seed all --profile=meta      # DEMO-14 prep
```

Each command is idempotent: re-running against already-seeded stores is a no-op with a summary line.
